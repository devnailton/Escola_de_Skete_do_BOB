Visão Geral

O Sistema de Gestão para Escola de Skate é uma aplicação web desenvolvida para gerenciar todos os aspectos operacionais de uma escola de skate, incluindo alunos, professores, turmas, matrículas e presenças.

O sistema foi projetado com uma arquitetura modular, utilizando Django como framework principal, seguindo as melhores práticas de desenvolvimento e segurança.

Principais funcionalidades:

  • Gestão de usuários com diferentes níveis de acesso
  • Cadastro e gerenciamento de alunos e responsáveis
  • Criação e administração de turmas
  • Controle de matrículas
  • Registro de presenças
  • Sistema de mensagens e contato
  • Feature flags para controle de funcionalidades

Modelo de Dados

O sistema utiliza o seguinte modelo de dados:

Usuario

Estende o modelo AbstractBaseUser do Django para autenticação personalizada.

  • email: Email único do usuário (usado para login)
  • papel: Função do usuário no sistema
  • ativo: Status de ativação da conta
  • criado_em: Data de criação
  • atualizado_em: Data da última atualização

Pessoa

Armazena informações pessoais dos indivíduos no sistema.

  • usuario: Relação OneToOne com Usuario
  • nome: Nome completo
  • cpf: CPF
  • telefone: Número de telefone
  • data_nascimento: Data de nascimento
  • endereco: Endereço completo
  • criado_em: Data de criação
  • atualizado_em: Data da última atualização

Aluno

Representa um aluno da escola de skate.

  • pessoa: Relação ForeignKey com Pessoa
  • responsavel: Relação ForeignKey com Pessoa (responsável)
  • data_matricula: Data da primeira matrícula
  • status: Status atual do aluno

Turma

Representa uma turma da escola.

  • nome: Nome da turma
  • faixa_etaria: Faixa etária dos alunos
  • professor: Relação ForeignKey com Pessoa (professor)
  • dias_da_semana: Dias em que ocorrem as aulas
  • horario_inicio: Horário de início das aulas
  • horario_fim: Horário de término das aulas
  • local: Local onde ocorrem as aulas

Matricula

Representa a matrícula de um aluno em uma turma.

  • aluno: Relação ForeignKey com Aluno
  • turma: Relação ForeignKey com Turma
  • data_inicio: Data de início da matrícula
  • data_fim: Data de término da matrícula (opcional)

Presenca

Registra a presença de um aluno em uma aula.

  • matricula: Relação ForeignKey com Matricula
  • data_aula: Data da aula
  • status: Status da presença (presente, ausente, justificado)

Funcionalidade

Controla as feature flags do sistema.

  • usuario: Relação ForeignKey com Usuario
  • papel: Papel do usuário
  • flag: Nome da funcionalidade
  • habilitado: Status de ativação
  • criado_em: Data de criação

MensagemContato

Armazena mensagens enviadas pelo formulário de contato.

  • nome: Nome do remetente
  • email: Email do remetente
  • mensagem: Conteúdo da mensagem
  • enviada_em: Data de envio

MensagemAutorizada

Controla o acesso às mensagens de contato.

  • mensagem: Relação ForeignKey com MensagemContato
  • usuario: Relação ForeignKey com Usuario
  • visualizada: Status de visualização

API

O sistema disponibiliza uma API RESTful para integração com outros sistemas. Todos os endpoints requerem autenticação via token JWT.

Endpoints de Usuários

  • GET /api/usuarios/ - Lista todos os usuários
  • POST /api/usuarios/ - Cria um novo usuário
  • GET /api/usuarios/{id}/ - Obtém detalhes de um usuário
  • PUT /api/usuarios/{id}/ - Atualiza um usuário
  • DELETE /api/usuarios/{id}/ - Remove um usuário

Endpoints de Alunos

  • GET /api/alunos/ - Lista todos os alunos
  • POST /api/alunos/ - Cria um novo aluno
  • GET /api/alunos/{id}/ - Obtém detalhes de um aluno
  • PUT /api/alunos/{id}/ - Atualiza um aluno
  • DELETE /api/alunos/{id}/ - Remove um aluno

Endpoints de Turmas

  • GET /api/turmas/ - Lista todas as turmas
  • POST /api/turmas/ - Cria uma nova turma
  • GET /api/turmas/{id}/ - Obtém detalhes de uma turma
  • PUT /api/turmas/{id}/ - Atualiza uma turma
  • DELETE /api/turmas/{id}/ - Remove uma turma
  • GET /api/turmas/{id}/alunos/ - Lista alunos de uma turma

Endpoints de Matrículas

  • GET /api/matriculas/ - Lista todas as matrículas
  • POST /api/matriculas/ - Cria uma nova matrícula
  • GET /api/matriculas/{id}/ - Obtém detalhes de uma matrícula
  • PUT /api/matriculas/{id}/ - Atualiza uma matrícula
  • DELETE /api/matriculas/{id}/ - Remove uma matrícula

Endpoints de Presenças

  • GET /api/presencas/ - Lista todas as presenças
  • POST /api/presencas/ - Registra uma nova presença
  • GET /api/presencas/{id}/ - Obtém detalhes de uma presença
  • PUT /api/presencas/{id}/ - Atualiza uma presença
  • DELETE /api/presencas/{id}/ - Remove uma presença
  • GET /api/presencas/turma/{turma_id}/data/{data}/ - Lista presenças de uma turma em uma data

Endpoints de Mensagens

  • GET /api/mensagens/ - Lista todas as mensagens
  • POST /api/mensagens/ - Envia uma nova mensagem
  • GET /api/mensagens/{id}/ - Obtém detalhes de uma mensagem
  • PUT /api/mensagens/{id}/visualizar/ - Marca uma mensagem como visualizada
  • PUT /api/mensagens/{id}/responder/ - Marca uma mensagem como respondida

Fluxos de Acesso

O sistema possui diferentes fluxos de acesso baseados no papel do usuário:

Administrador

Acesso completo a todas as funcionalidades do sistema:

  • Gerenciamento de usuários
  • Gerenciamento de alunos e responsáveis
  • Gerenciamento de turmas
  • Gerenciamento de matrículas
  • Registro e consulta de presenças
  • Configuração de feature flags
  • Acesso a todas as mensagens de contato
  • Geração de relatórios

Professor

Acesso limitado às suas turmas e alunos:

  • Visualização das suas turmas
  • Visualização dos alunos das suas turmas
  • Registro e consulta de presenças das suas turmas
  • Visualização de informações básicas dos alunos

Aluno/Responsável

Acesso limitado às suas próprias informações:

  • Visualização das suas matrículas
  • Visualização das suas turmas
  • Visualização do seu histórico de presenças
  • Atualização de informações pessoais
  • Envio de mensagens

Visitante

Acesso apenas à área pública do site:

  • Visualização da landing page
  • Envio de mensagens pelo formulário de contato

Feature Flags

O sistema utiliza feature flags para controlar o acesso a funcionalidades específicas. As flags são configuradas por usuário e papel.

Flag Descrição Papéis Padrão
relatorios_avancados Acesso a relatórios avançados e estatísticas Administrador
gerenciamento_usuarios Criação e edição de usuários Administrador
gerenciamento_turmas Criação e edição de turmas Administrador
registro_presencas Registro de presenças Administrador, Professor
mensagens_contato Acesso às mensagens de contato Administrador
area_aluno Acesso à área do aluno Aluno, Responsável

Segurança

O sistema implementa as seguintes medidas de segurança:

  • Autenticação: Sistema de login seguro com senhas hash
  • Autorização: Controle de acesso baseado em papéis e feature flags
  • Proteção CSRF: Tokens CSRF em todos os formulários
  • Proteção XSS: Escape automático de conteúdo em templates
  • Proteção SQL Injection: Uso do ORM Django para consultas seguras
  • HTTPS: Recomendado para produção
  • Rate Limiting: Limitação de requisições para prevenir ataques de força bruta
  • Validação de Dados: Validação rigorosa de todos os inputs

Execução

Para executar o sistema em ambiente de desenvolvimento:

  1. Clone o repositório
  2. Crie e ative um ambiente virtual Python
  3. Instale as dependências: pip install -r requirements.txt
  4. Configure o banco de dados: python manage.py migrate
  5. Crie um superusuário: python manage.py createsuperuser
  6. Execute o servidor: python manage.py runserver

Para ambiente de produção, recomenda-se:

  • Usar Gunicorn ou uWSGI como servidor WSGI
  • Configurar Nginx como proxy reverso
  • Usar PostgreSQL como banco de dados
  • Configurar HTTPS com certificado SSL
  • Implementar backup regular do banco de dados
